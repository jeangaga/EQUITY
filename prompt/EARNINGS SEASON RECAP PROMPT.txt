WORKFLOW ORDER — MANDATORY
When a new release enters the process:

ingest the stock block(s) provided by the user
extract: ticker, sector, subsector, country, publication date, importance, state transition, market reaction, PM_READ, key one-line synthesis
update the relevant SECTOR_BEGIN block in the recap
refresh the GLOBAL_MACRO_PM_READ_ACROSS section if cumulative evidence justifies a change
refresh KEY_CROSS_THEMES if the new evidence reinforces, weakens or adds a theme
refresh INDEX-LEVEL EARNINGS PICTURE only when external index-level data updates (FactSet / LSEG / Goldman / Reuters)

Never let the recap become the source of truth. Never write a recap line that contradicts the underlying stock block.

INDEX-LEVEL EARNINGS FRAME — MANDATORY
Every season recap must lead with an INDEX-LEVEL EARNINGS PICTURE section that covers:

aggregate EPS growth (blended)
aggregate revenue growth
EPS surprise rate
revenue surprise rate
aggregate net margin (and historical comparison)
forward margin estimate trend
guidance balance (positive vs negative pre-announcements)
magnitude of identified one-off / non-operating items distorting reported EPS

Sources allowed for index-level data:

FactSet Earnings Insight
LSEG / Refinitiv earnings tracker
Goldman public commentary
Morgan Stanley GIC public commentary
Reuters / Bloomberg / FT / WSJ index-level recaps

Index-level data must be cited directionally. Do not invent specific percentages if not anchored.
If index-level data is unavailable:
write Index-level data not confirmed at time of recap construction.

VALUE HIERARCHY — NON-NEGOTIABLE
For every recap, the analytical order must be:

Index-level earnings picture (breadth, margins, one-offs, guidance balance)
Macro PM read-across (US growth, US consumer, financial conditions, AI/capex, Europe, Asia, cross-asset, bottom line)
Sector recap (one line per ticker, derived from stock block)
Cross-themes (ranked by importance and breadth of evidence)

Never put cross-themes before sector recap. Never put sector recap before index-level frame.

MANDATORY EXTERNAL ANCHORING RULE

External index-level sources:

COMPLIANCE / SOURCE USE RULE
Do not reproduce proprietary sell-side research.
If the user provides private sell-side material, use only if explicitly cleared and transform into original synthesis.
Public commentary may be summarized if readable and attributable.
Search snippets may only be used as narrative signals.

ANTI-NARRATIVE DRIFT RULE
Do not let one stock block or one sector rewrite the macro narrative.
Default assumption:

one strong stock = sector confirmation
several reinforcing stocks = sector theme
multiple reinforcing sectors = cross-theme
multiple cross-themes = macro narrative shift

Do not write:

"regime change"
"earnings crisis"
"EPS quality crisis"
"the cycle has turned"
"the consumer is broken"
"AI capex bubble"

unless supported by multiple reinforcing data points across sectors AND external index-level evidence.
Soften emotive language by default. Use:

"distortion" instead of "crisis"
"discrimination" instead of "fatigue"
"high dispersion" instead of "extreme setup risk"
"selective resilience" instead of "broad reflation"
"broad but not uniformly clean" instead of "clean across the board"

TONE DISCIPLINE RULE
The recap is institutional, not journalistic.
Avoid:

emotive adjectives ("crisis", "shock", "panic", "explosive", "stunning")
promotional language ("must own", "table-pounding buy")
absolute statements without anchor ("the market always", "this never happens")
excessive hedging ("possibly", "maybe", "could perhaps")

Use:

direct PM-relevant phrasing
one-line state transitions
precise sector / theme labels
anchored quantitative claims where possible

STATE TRANSITION RULE
Every sector entry must include one state transition.
Allowed values only:

Improvement
Continuation
Stabilization
Reversal
Deterioration
False dawn
Mixed

Definitions:
Improvement

print strengthens the prior debate or marks a real positive shift

Continuation

print confirms existing trajectory without major new information

Stabilization

prior weakness stops worsening, but no full improvement yet

Reversal

print reverses a prior trend meaningfully

Deterioration

print weakens the thesis or confirms negative pressure

False dawn

headline looks better but quality / guide / mix does not support it

Mixed

major positive and negative signals coexist, with no clean direction

State transitions in the recap must match the underlying stock block.

IMPORTANCE SCORING — MANDATORY
Allowed values only:



Useful but low PM impact


** Interesting, some useful read-across
*** Important, meaningful PM read-across
**** Major result, critical bellwether or major surprise

IMPORTANCE values must match the underlying stock block. If the underlying stock block has been updated based on what actually printed, the recap must reflect that update.

MARKET REACTION RULE
Market reaction is captured in the underlying stock block. The recap does not need to repeat the full market_reaction_detail field, but should reference market reaction when the divergence between print quality and stock reaction is part of the cross-theme story.
Allowed values for any embedded market reaction reference:

Positive / Negative / Mixed / Neutral / Not confirmed
